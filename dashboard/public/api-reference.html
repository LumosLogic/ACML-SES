Section 1

API Reference

The ACML SES API lets you send transactional and bulk email programmatically via a single HTTP endpoint. All requests use multipart/form-data and are authenticated with your API key.

POST https://lumosmails.lumoslogic.com/api/send

Authentication: X-Api-Key: <your-api-key>  ·  Content-Type: multipart/form-data

Request Fields

Field Type Required Description
from string No Sender name + email, e.g. ACML <ACML@mail.acml.co.in>. Defaults to system default if omitted. Must match your allowed domain.
to string Yes* Comma-separated recipient email addresses. *Required if neither recipients nor csv is provided.
recipients string (JSON) Yes* JSON array with per-recipient variables: [{"email":"a@x.com","vars":{"name":"John"}}]
csv file Yes* CSV file with an email column. Extra columns become per-recipient template variables.
subject string Yes Email subject line. Supports {{variable}} placeholders.
body string Yes Email body (HTML or plain text). Supports {{variable}} and {{unsubscribe_url}}.
isHtml string No "true" or "false". Default: "true"
replyTo string No Reply-To email address.
cc string No Comma-separated CC email addresses.
bcc string No Comma-separated BCC email addresses.
globalVars string (JSON) No JSON object applied to all recipients, e.g. {"company":"ACML"}. Per-recipient vars override these.
send_at string No ISO 8601 datetime to schedule delivery, e.g. 2026-07-01T09:00:00Z
attachments file(s) No Up to 3 files, max 2 MB each. Allowed: PDF, JPG, PNG, GIF, WEBP, DOC, DOCX, XLS, XLSX, TXT, CSV, ZIP.
callback_url string No Webhook URL called when a bulk job completes.

Response Format

The API returns JSON. The shape depends on whether the send is immediate or queued/scheduled.

Standard Response

JSON 
{
  "status": "sent | partial | failed | queued | scheduled | skipped",
  "summary": {
    "total_requested":    10,
    "sent":               8,
    "failed":             0,
    "suppressed":         2,
    "blocked_by_daily_limit": 0
  },
  "warning": "Daily limit of 5000 emails reached. 2 email(s) were not sent. Resets at midnight IST.",
  "results": [
    {
      "email":     "user@example.com",
      "status":    "sent",
      "messageId": "abc123"
    }
  ]
}

Bulk / Scheduled Response >50 recipients or send_at set

JSON 
{
  "status":            "queued | scheduled",
  "job_id":           "42",
  "total_recipients": 500,
  "skipped":          3,
  "poll_url":         "/api/job-status/42",
  "send_at":          "2026-07-01T09:00:00Z"
}

Test Scenarios

Eleven ready-to-run tests covering the full range of API features. Click a card to expand it and copy the curl command.

Sends a basic plain text email with no HTML or attachments. The simplest possible API call.

Expected: {"status":"sent","summary":{"sent":1,...}}
Shell — curl 
curl -X POST https://lumosmails.lumoslogic.com/api/send \
  -H "X-Api-Key: YOUR_API_KEY" \
  --form-string "from=ACML <ACML@mail.acml.co.in>" \
  --form-string "to=you@example.com" \
  --form-string "subject=Test 1 - Simple Email" \
  --form-string "isHtml=false" \
  --form-string "body=Hello, this is a simple plain text email. No HTML, no attachments."

Section 3

Error Reference

All error responses return JSON with an error field describing the issue. The HTTP status code indicates the error category.

Status Error Message Cause
400 "subject" is required The subject field was not included in the request.
400 "body" is required The body field was not included in the request.
400 Sender domain is not allowed The domain in the from field does not match the allowed domain for your API key.
400 Too many recipients The number of recipients in a single request exceeded the maximum allowed limit.
400 Invalid CSV The uploaded CSV file is missing headers or does not contain an email column.
400 File type not allowed An attachment's MIME type is not in the permitted list (PDF, JPG, PNG, GIF, WEBP, DOC/X, XLS/X, TXT, CSV, ZIP).
401 Unauthorized The X-Api-Key header is missing, empty, or the key is invalid / revoked.
429 You have exceeded your limit Your daily email quota is fully exhausted. Quota resets at midnight IST (Asia/Kolkata).
500 Internal Server Error All emails in the request failed to send due to an upstream delivery error.
Section 4

Notes

⇉

Bulk sending: Requests with more than 50 recipients are automatically queued as background jobs. The response returns a job_id instead of immediate results. Poll /api/job-status/{job_id} to track progress and retrieve final delivery stats.

{}

Template variables: Use {{variable_name}} placeholders anywhere in the subject or body. At send time they are replaced with values from each recipient's vars object (from recipients or CSV columns). Values from globalVars serve as defaults and are overridden by per-recipient vars.

↩

Unsubscribe: Add {{unsubscribe_url}} anywhere in the email body to automatically inject a unique, per-recipient unsubscribe link. Recipients who click it are suppressed from future sends.

📎

Attachments: Maximum of 3 files per request, each up to 2 MB. Allowed file types: PDF, JPG, PNG, GIF, WEBP, DOC, DOCX, XLS, XLSX, TXT, CSV, ZIP. Files exceeding size or type restrictions will result in a 400 error.

⌛

Daily limit: Each API key has a daily email send quota. The quota resets at midnight IST (Asia/Kolkata). Partial sends are allowed — emails up to the remaining quota go through, and any beyond are blocked with a blocked_by_daily_limit count in the response. Once fully exhausted, the API returns HTTP 429 until reset.

🚫

Suppression: Addresses that have previously hard-bounced or generated a spam complaint are automatically suppressed. They are skipped silently during sending and counted in the suppressed field of the response summary — no error is raised.
